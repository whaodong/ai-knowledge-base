## 推荐方案

### 场景1：大规模数据处理（成本优先）
**推荐：** Ollama + bge-large-zh（本地）或 text-embedding-3-small（云端）

**配置：**
```yaml
embedding:
  strategy: COST_FIRST
  current-model: bge-large-zh
  fallback-models:
    - text-embedding-3-small
```

### 场景2：高精度匹配（质量优先）
**推荐：** text-embedding-3-large 或 bge-large-zh

**配置：**
```yaml
embedding:
  strategy: QUALITY_FIRST
  current-model: text-embedding-3-large
  fallback-models:
    - bge-large-zh
```

### 场景3：实时处理（速度优先）
**推荐：** Ollama + m3e-base（本地部署）

**配置：**
```yaml
embedding:
  strategy: SPEED_FIRST
  current-model: m3e-base
  ollama:
    enabled: true
    base-url: http://localhost:11434
```

### 场景4：国内部署（合规优先）
**推荐：** Alibaba text-embedding-v3

**配置：**
```yaml
embedding:
  strategy: FIXED
  current-model: text-embedding-v3
  alibaba:
    enabled: true
    api-key: ${ALIBABA_API_KEY}
```

### 场景5：混合部署（平衡模式）
**推荐：** 本地+云端混合

**配置：**
```yaml
embedding:
  strategy: BALANCED
  current-model: bge-large-zh
  fallback-models:
    - text-embedding-v3
    - text-embedding-3-small
  ollama:
    enabled: true
  alibaba:
    enabled: true
```

## 批量处理优化

### 配置建议

```yaml
embedding:
  batch:
    enabled: true
    default-batch-size: 20      # 根据文本长度调整
    max-batch-size: 100         # 最大批大小
    parallel-threads: 4         # 并行线程数
    retry-times: 3              # 失败重试次数
    timeout: 30000              # 超时时间
```

### 性能提升

- **动态批大小：** 根据文本长度自动调整，长文本减小批大小
- **并行处理：** 多线程并发编码，提升吞吐量
- **错误重试：** 失败自动重试，提高成功率
- **缓存优化：** 相同文本不重复编码，节省成本

## 缓存策略

### 配置建议

```yaml
embedding:
  cache:
    enabled: true
    expire-seconds: 86400      # 24小时过期
    key-prefix: "embedding:cache:"
    max-size: 100000            # 最大缓存数
    track-hit-rate: true        # 统计命中率
```

### 缓存效果

| 场景 | 命中率 | 成本节省 |
|------|--------|---------|
| 文档问答 | 70-90% | 70-90% |
| 知识库检索 | 60-80% | 60-80% |
| 实时对话 | 30-50% | 30-50% |

## 监控指标

### 关键指标

1. **编码延迟：** P50、P95、P99
2. **吞吐量：** QPS
3. **成功率：** 编码成功率
4. **缓存命中率：** 缓存效果
5. **成本统计：** API调用次数

### Prometheus 监控

```yaml
# 编码延迟
embedding_duration_seconds{model="text-embedding-3-small",quantile="0.99"}

# 编码次数
embedding_requests_total{model="text-embedding-3-small",status="success"}

# 缓存命中率
embedding_cache_hit_rate
```

## 最佳实践

1. **启用缓存：** 大幅降低成本和延迟
2. **批量处理：** 提升吞吐量，降低API调用次数
3. **降级策略：** 主模型失败时自动切换备用模型
4. **本地优先：** 隐私要求高的场景使用本地模型
5. **监控告警：** 设置延迟和成功率告警

## 总结

选择编码器需要综合考虑：
- **成本预算：** 本地免费 vs 云端付费
- **质量要求：** 高精度 vs 快速处理
- **隐私合规：** 数据是否可以上传云端
- **部署环境：** 是否有GPU资源
- **访问范围：** 国内还是国际访问

建议采用**混合部署**策略，本地模型处理常规请求，云端模型作为降级备选，既保证性能又控制成本。
