# Embedding Service - 多编码器支持

## 概述

Embedding Service 提供了强大的多编码器支持，允许灵活切换不同的向量编码模型，支持批量处理、缓存优化和智能降级。

## 核心功能

### 1. 多编码器支持

支持以下编码器：

| 提供者 | 模型 | 维度 | 特点 |
|--------|------|------|------|
| **OpenAI** | text-embedding-3-small | 1536 | 快速、低成本 |
| **OpenAI** | text-embedding-3-large | 3072 | 高质量 |
| **Ollama** | bge-large-zh | 1024 | 本地免费、中文优化 |
| **Ollama** | m3e-base | 768 | 本地快速 |
| **Alibaba** | text-embedding-v3 | 1024 | 国内快速 |
| **Baidu** | Embedding-V1 | 1024 | 百度生态 |

### 2. 智能模型选择

支持5种选择策略：

- **FIXED**: 固定模型（默认）
- **COST_FIRST**: 成本优先，选择免费或低成本模型
- **QUALITY_FIRST**: 质量优先，选择高质量模型
- **SPEED_FIRST**: 速度优先，选择处理速度快的模型
- **BALANCED**: 平衡模式，综合考虑各因素

### 3. 批量编码优化

- 动态批大小调整
- 并行多线程处理
- 错误重试机制
- 进度实时追踪

### 4. 编码缓存

- Redis 持久化缓存
- 自动过期机制
- 命中率统计
- 批量缓存操作

### 5. 自动降级

主模型失败时自动切换备用模型，确保服务可用性。

## 快速开始

### 配置

在 `application.yml` 中配置：

```yaml
embedding:
  # 当前使用的编码器
  current-model: text-embedding-3-small
  
  # 选择策略
  strategy: FIXED
  
  # 备用模型（降级用）
  fallback-models:
    - bge-large-zh
    - text-embedding-v3
  
  # OpenAI 配置
  openai:
    api-key: ${OPENAI_API_KEY}
  
  # Ollama 本地模型配置
  ollama:
    enabled: true
    base-url: http://localhost:11434
  
  # 批量处理配置
  batch:
    enabled: true
    default-batch-size: 20
    parallel-threads: 4
  
  # 缓存配置
  cache:
    enabled: true
    expire-seconds: 86400
```

### API 使用

#### 1. 单文本编码

```bash
POST /api/v1/embeddings
Content-Type: application/json

{
  "text": "这是一段需要编码的文本",
  "model": "text-embedding-3-small"  // 可选
}
```

响应：
```json
{
  "code": 200,
  "message": "向量化成功",
  "data": {
    "taskId": "uuid",

    "dimension": 1536,

    "status": "COMPLETED",
    "duration": 50

}
```

## 核心类说明

### EmbeddingModelRegistry

编码器注册中心，管理所有编码器提供者。

```java
// 获取可用模型列表
List<EmbeddingModelType> models = registry.getAvailableModels();

// 根据策略选择最佳模型
EmbeddingModelType model = registry.selectBestModel(EmbeddingStrategy.QUALITY_FIRST);

// 执行编码（支持降级）
EmbeddingResult result = registry.embedWithFallback(text, preferredModel);

// 获取提供者状态
Map<String, Boolean> status = registry.getProviderStatus();
```

### BatchEmbeddingService

批量编码服务，提供高性能批量处理能力。

```java
// 同步批量编码
BatchResult result = batchService.batchEmbed(texts, modelType);

// 指定批大小
BatchResult result = batchService.batchEmbed(texts, modelType, 50);

// 异步批量编码
CompletableFuture<BatchResult> future = batchService.batchEmbedAsync(texts, modelType);

// 并行批量编码
BatchResult result = batchService.batchEmbedParallel(texts, modelType, 4);
```

### EmbeddingCacheService

编码缓存服务，提供Redis持久化缓存。

```java
// 获取缓存
EmbeddingResult cached = cacheService.get(text, modelType);

// 存入缓存
cacheService.put(text, modelType, result);

// 获取统计
CacheStats stats = cacheService.getStats();

// 清空缓存
cacheService.clearAll();
```

## 性能优化建议

### 1. 启用缓存

缓存可以大幅降低成本和延迟：

```yaml
embedding:
  cache:
    enabled: true
    expire-seconds: 86400  # 根据数据更新频率调整
```

### 2. 批量处理

使用批量接口而不是多次单个调用：

```java
// ✅ 推荐
List<String> texts = Arrays.asList("文本1", "文本2", "文本3");
batchService.batchEmbed(texts, modelType);

// ❌ 不推荐
for (String text : texts) {
    embedService.embedText(text);
}
```

### 3. 选择合适的模型

根据场景选择模型：

- **大规模处理**: text-embedding-3-small 或本地模型
- **高精度要求**: text-embedding-3-large
- **实时处理**: m3e-base（本地）
- **隐私要求高**: 本地模型

### 4. 配置降级策略

确保服务可用性：

```yaml
embedding:
  current-model: text-embedding-3-small
  fallback-models:
    - bge-large-zh      # 本地备用
    - text-embedding-v3  # 云端备用
```

## 监控指标

建议监控以下指标：

1. **编码延迟**: P50、P95、P99
2. **吞吐量**: QPS
3. **成功率**: 编码成功率
4. **缓存命中率**: 缓存效果
5. **模型使用分布**: 各模型使用比例

Prometheus 示例：

```yaml
# 编码延迟
embedding_duration_seconds{model="text-embedding-3-small",quantile="0.99"}

# 编码次数
embedding_requests_total{model="text-embedding-3-small",status="success"}

# 缓存命中率
embedding_cache_hit_rate
```

## 故障排查

### 1. 模型不可用

检查提供者配置和健康状态：

```bash
GET /api/v1/embeddings/providers/status
```

### 2. 编码失败

查看日志中的错误信息，常见原因：

- API Key 无效或过期
- 网络连接问题
- 模型名称错误
- 配额不足

### 3. 缓存未生效

检查：

- Redis 是否正常运行
- 缓存配置是否启用
- 文本是否完全相同（包括空格）

### 4. 批量处理慢

优化建议：

- 增加并行线程数
- 调整批大小
- 启用缓存
- 使用本地模型

```yaml
    parallel-threads: 8

    enabled: true

```
